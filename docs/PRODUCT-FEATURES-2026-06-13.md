# 产品功能点与前端页面验证清单（2026-06-13）

本文档基于当前代码中的导航配置、页面路由、前端交互、领域类型与接口调用关系整理，目标是给前端页面验证提供统一口径。

验证时建议按角色分别登录，逐页核对「入口是否可见」「数据是否按权限展示」「按钮是否按状态启用」「提交后是否刷新列表/详情」「错误提示是否清晰」。

## 一、角色与导航入口

| 角色 | 导航入口 | 核心定位 |
| --- | --- | --- |
| 管理员 Admin | 工作台、待我处理、项目治理、用户管理、注册审批、绑定关系、系统参数、审计日志、统计报表、通知中心、AI 助手、个人设置 | 管理平台用户、审批、绑定、参数、审计、项目治理与全局报表 |
| 编辑 Editor | 工作台、待我处理、SI 选题策划库、预发记录、我的项目、审稿工作台、通知中心、AI 助手、个人设置 | 创建与管理 SI，预发 SI，转项目，审稿，推进项目 |
| 作者 Author | 工作台、待我处理、我的 SI、我的项目、通知中心、个人设置 | 查看预发 SI，参与项目 Doc 写作，提交稿件，处理退回 |
| 未登录用户 | 登录、注册、忘记密码、账号状态 | 完成登录、作者注册、密码找回申请、查看账号审批/禁用状态 |

前端验证重点：

- 不同角色只显示对应导航项。
- 移动端侧边栏、桌面端侧边栏、顶部用户区、退出登录入口保持一致。
- 无权限访问页面时应出现权限提示或被服务端/接口拒绝，不能展示越权数据。
- 未登录访问业务页应回到登录页或被鉴权层拦截。

## 二、核心业务状态口径

| 业务对象 | 状态 | 验证说明 |
| --- | --- | --- |
| 用户 | `pending`、`active`、`disabled`、`rejected` | 注册后进入待审批；审批通过后可登录；禁用/拒绝账号不可进入业务页 |
| SI | `draft`、`prereleased`、`converted`、`archived` | 草稿/预发可编辑；已转项目不可再预发；归档当前页面入口存在但禁用 |
| SI 预发记录 | `active`、`withdrawn`、`converted` | 作者只看 active/converted；编辑可收回 active；converted 可进入项目 |
| 项目生命周期 | `draft`、`active`、`completed`、`archived`、`cancelled` | completed/archived/cancelled 为只读态 |
| 项目阶段 | `synopsis`、`outline`、`chapter`、`release`、`completed` | 梗概通过后进入细纲，细纲通过后进入正文，全部正文通过后可解锁质检 |
| Doc 状态 | `draft`、`submitted`、`returned`、`approved` | draft 可编辑/提交；submitted 可审核；returned 由作者修改；approved 作为阶段通过依据 |
| Doc 持有人 | `author`、`editor`、`none` | 决定当前稿件由谁处理，前端需展示正确持有人 |
| 阶段计划 | `not_started`、`in_progress`、`due_soon`、`overdue`、`completed` | 项目列表、详情、章节页、治理页均需展示计划状态 |
| 质检 Doc | `locked`、`unlocked`、`draft`、`submitted`、`returned`、`approved` | locked 不可进入质检 Doc；解锁后可进入完整 Doc 流程 |

## 三、登录前功能

| 页面 | 路径 | 功能点 | 验证重点 |
| --- | --- | --- | --- |
| 登录 | `/login` | 邮箱/密码登录、账号状态拦截、登录成功进入业务区 | 错误账号提示；pending/rejected/disabled 跳转或提示；登录后按角色进入正确页面 |
| 注册 | `/register` | 作者注册申请、密码确认、基础信息提交 | 只能申请作者账号；提交后进入待审批；重复邮箱应提示 |
| 忘记密码 | `/forgot-password` | 邮箱提交密码找回申请 | 成功提示不暴露账号是否存在；管理员端应产生待处理通知/待办 |
| 账号状态 | `/account-status` | 展示待审批、拒绝、禁用等状态说明 | 文案与状态一致；能引导用户返回登录或重新处理 |

## 四、登录后公共框架

| 模块 | 功能点 | 验证重点 |
| --- | --- | --- |
| 应用布局 | 侧边栏、顶部栏、面包屑、页面标题、操作区 | 不同屏宽下不遮挡内容；菜单高亮正确；页面标题与当前路径一致 |
| 角色上下文 | 当前用户、角色、导航权限 | 刷新页面后仍能恢复用户态；不同角色不能看到不属于自己的入口 |
| 通知提示 | 顶部通知入口、未读数量、通知中心跳转 | 新通知产生后未读数应更新；已读后数量减少 |
| 退出登录 | 清理会话并返回登录页 | 退出后不能通过返回键看到敏感数据 |
| 首次/强制改密 | 账号存在 `mustChangePassword` 时应引导改密 | 当前需重点核对是否全链路生效，见“已知待闭环项” |

## 五、工作台、待办、通知、报表

### 5.1 工作台

| 角色 | 页面 | 数据来源 | 功能点 |
| --- | --- | --- | --- |
| 管理员 | `/dashboard` | `/api/admin/dashboard` | 用户数、项目数、审批、审计/平台统计摘要 |
| 编辑 | `/dashboard` | `/api/dashboard` | 负责项目、待审 Doc、退回 Doc、预警/逾期项目、近期活动 |
| 作者 | `/dashboard` | `/api/dashboard` | 我的项目、待修改 Doc、阶段预警、预发 SI/近期活动 |

验证重点：

- 角色切换后看板指标不同。
- 卡片跳转目标正确。
- 空数据、加载中、接口失败状态可见。

### 5.2 待我处理

路径：`/todos`

功能点：

- 读取真实待办列表：`/api/todos`。
- 分类切换：全部、Doc 待审、退回待改、注册审批。
- 注册审批分类仅管理员可见。
- 展示任务类型、业务对象、发起人、截止时间、创建时间、已读时间。
- 支持批量标记已读：`POST /api/todos/read-all`。
- 每条待办提供“处理”按钮，跳转到对应业务页。

验证重点：

- 管理员能看到注册审批待办，编辑/作者不能看到。
- 未读待办有视觉标识。
- 批量已读后列表状态即时更新。
- 跳转目标与待办类型一致。

### 5.3 通知中心

路径：`/notifications`

功能点：

- 读取通知列表：`/api/notifications`。
- 支持已读状态筛选：全部、未读、已读。
- 支持通知类型筛选：Doc 提交、Doc 退回、Doc 通过、注册审批、忘记密码等。
- 点击通知打开右侧详情抽屉。
- 打开未读通知时自动标记单条已读：`POST /api/notifications/:id/read`。
- 支持全部已读：`POST /api/notifications/read-all`。
- 通知详情可跳转相关业务内容。

验证重点：

- 未读数量、未读标识、筛选结果一致。
- 打开详情后当前通知变为已读。
- 抽屉关闭、跳转、移动端宽度表现正常。

### 5.4 统计报表

路径：`/reports`

功能点：

- 时间范围切换：近 7 天、近 30 天、近 90 天、全部时间。
- 管理员报表：用户数、项目总数、已完成项目、逾期项目、总提交字数、今日提交、今日审核、今日退回、阶段项目数、作者提交排行、编辑效率排行。
- 编辑报表：负责项目数、待审核 Doc、退回 Doc、即将到期项目、逾期项目、最近处理记录。
- 作者报表：我的项目数、提交/退回/阶段相关统计、近期记录。

接口：

- 管理员：`/api/admin/reports?range=...`
- 编辑/作者：`/api/reports?range=...`

当前验证口径：

- 时间范围会触发接口刷新。
- 页面存在“统计维度”选择，但当前前端仅保存本地状态，没有参与接口查询或图表切换，验证时应作为待闭环项记录。

## 六、SI 选题策划

### 6.1 编辑端 SI 选题策划库

路径：`/si`

功能点：

- 读取 SI 列表：`/api/si`。
- 读取启用主类型：`/api/si-main-types`，失败时回退默认主类型。
- 支持搜索 SI 标题、Trope。
- 支持按 SI 状态筛选。
- 支持按主类型筛选。
- 展示标题、状态、主类型、Trope、适配作者、预发数量、更新时间、已转项目标识。
- 支持进入详情、编辑、预发、版本历史。

当前限制：

- “归档”和“删除”按钮当前为禁用状态，页面有入口但未接入操作。
- 已转项目 SI 的预发按钮禁用。

### 6.2 新建/编辑 SI

路径：`/si/new`、`/si/:id/edit`

功能点：

- 填写 SI 标题、主类型、Trope、卖点、故事梗概、目标读者、节奏/风格等策划字段。
- 选择适配作者。
- 保存草稿。
- 保存并预发。
- 编辑已存在 SI。
- 主类型来自管理员参数配置。

验证重点：

- 必填字段校验准确。
- 保存后能在列表和详情看到最新内容。
- “保存并预发”应打开或完成预发流程，且预发记录可追踪。
- 草稿/预发状态可编辑，已转项目/归档态不可编辑。

### 6.3 SI 详情

路径：`/si/:id`

功能点：

- 展示 SI 完整策划内容。
- 展示绑定/适配作者与预发记录。
- 支持编辑。
- 支持预发给作者。
- 支持收回 active 预发记录。
- 支持将预发记录转换为项目。
- 已转换记录可跳转项目详情。

验证重点：

- 转项目时必须选择有效作者/预发记录。
- 转项目后 SI 状态、预发记录状态、项目列表均应同步变化。
- 收回后作者端“我的 SI”不再显示该记录。

### 6.4 SI 版本历史

路径：`/si/:id/versions`

功能点：

- 展示 SI 保存形成的版本记录。
- 可查看版本快照内容。

当前限制：

- 页面存在回滚相关入口/按钮但当前不可用，虽然后端已有回滚接口痕迹；验证时应记录为产品待闭环项。

### 6.5 编辑端预发记录

路径：`/si/prereleases`

功能点：

- 汇总 SI 预发记录。
- 查看预发作者、预发状态、预发说明、时间。
- 对 active 记录执行收回。
- 对 active 记录执行转项目。
- converted 记录可进入项目。

验证重点：

- active、withdrawn、converted 状态显示准确。
- 收回后作者不可见。
- 转项目后不能重复转换。

### 6.6 作者端我的 SI

路径：`/my-si`、`/my-si/:recordId`

功能点：

- 作者查看编辑预发给自己的 SI。
- 支持搜索 SI 标题、Trope。
- 支持按 active/converted 状态筛选。
- 详情页查看完整 SI 内容、预发说明、预发编辑、预发时间。
- converted 记录可进入项目。

验证重点：

- 作者不能编辑、收回、删除 SI。
- withdrawn 记录不应出现在作者列表。
- converted 记录项目跳转正确。

## 七、项目协作

### 7.1 我的项目

路径：`/projects`

功能点：

- 编辑/作者查看自己参与的项目。
- 管理员也可从该入口查看有权限的项目视图。
- 支持搜索项目标题、来源 SI。
- 支持按当前阶段、生命周期、负责人、作者、是否逾期筛选。
- 展示项目标题、来源 SI、负责编辑、负责作者、当前阶段、生命周期、计划状态、待处理数量、最近更新。
- 编辑/管理员可导出项目 DOCX。

接口：

- `/api/projects`
- 导出：`/api/projects/:id/export?scope=project&format=docx`

验证重点：

- 编辑只能看到负责项目，作者只能看到分配项目，管理员按权限口径看到项目。
- 筛选为前端本地筛选，刷新后不保留筛选条件。
- 导出按钮只对编辑/管理员显示。

### 7.2 项目详情

路径：`/projects/:id`

功能点：

- 展示生命周期、当前阶段、负责编辑、负责作者、创建时间。
- 展示阶段进度。
- 展示阶段计划表。
- 右侧项目操作区集中提供可执行动作。
- 正文阶段可进入章节管理。
- 全部章节通过后，编辑/管理员可手动解锁质检。
- 质检 Doc 通过后，编辑/管理员可标记项目完成。
- 编辑/管理员可导出项目。
- completed/archived/cancelled 项目进入只读状态。

接口：

- 详情：`/api/projects/:id`
- 解锁质检：`POST /api/projects/:id/qc/unlock`
- 完成项目：`POST /api/projects/:id/complete`

验证重点：

- 按项目阶段展示对应操作。
- 未满足条件的按钮应禁用并保持可理解状态。
- 解锁/完成后页面刷新并展示成功提示。

### 7.3 阶段计划

路径：`/projects/:id/plan`

功能点：

- 展示项目阶段计划、起止日期、计划状态。
- 项目详情中也嵌入只读阶段计划表。

验证重点：

- 计划状态与列表、详情、章节页一致。
- 逾期/即将到期状态的颜色与文案一致。

### 7.4 正文章节管理

路径：`/projects/:id/chapters`

功能点：

- 展示正文总章节、已通过章节、阶段计划状态。
- 正文阶段未解锁时，提示只能查看章节结构。
- 管理角色可新增章节，输入章节标题和可选章节号。
- 章节列表展示排序、标题、Doc 状态、持有人、字数、最近交接说明、最近操作人和时间。
- 支持进入当前稿件、历史版本、阅读模式。
- 支持上移/下移重排章节。
- 支持删除未通过章节，删除前弹窗确认。

接口：

- 列表：`/api/projects/:id/chapters`
- 新增：`POST /api/projects/:id/chapters`
- 排序：`PATCH /api/projects/:id/chapters/order`
- 删除：`DELETE /api/projects/:id/chapters/:docId`

当前限制：

- “重排说明”按钮当前为禁用状态，仅作为提示入口。

### 7.5 质检

路径：`/projects/:id/qc`

功能点：

- 展示质检状态。
- 展示解锁条件：正文总章节数、已通过章节数、未通过章节数。
- 未通过章节列表展示标题与 Doc 状态。
- 编辑/管理员可在全部章节通过后手动解锁质检。
- 解锁后可进入质检 Doc。
- 质检 Doc 通过后可标记项目完成。
- 说明终稿导出优先取质检 Doc。

验证重点：

- 未全部通过章节时解锁按钮禁用。
- 解锁后质检 Doc 入口出现。
- 完成项目后项目生命周期变为 completed 且只读。

## 八、Doc 协作与审稿

### 8.1 当前稿件编辑页

路径：`/projects/:id/docs/:docType`

功能点：

- 读取当前 Doc：`/api/docs/:docId/current`。
- 展示 Doc 标题、状态、持有人、权限、当前内容、阶段计划摘要。
- 作者/编辑根据持有人与权限编辑内容。
- 保存草稿：`POST /api/docs/:docId/save`。
- 提交审核：`POST /api/docs/:docId/submit`。
- 退回作者：`POST /api/docs/:docId/return`。
- 审核通过：`POST /api/docs/:docId/approve`。
- 提交/退回/通过均带 `lockVersion`，用于并发版本校验。
- 编辑持稿时支持修订相关交互展示。
- 提供历史版本、阅读模式入口。

验证重点：

- 不同状态下按钮权限正确：草稿可保存/提交，已提交可审核，已通过不可继续编辑。
- 退回必须填写说明，通过可填写说明。
- 保存或流转后状态、持有人、待办、通知同步变化。
- 并发场景中旧 `lockVersion` 应被拒绝。

### 8.2 阅读模式/清洁版

路径：`/projects/:id/docs/:docType/clean`

功能点：

- 展示当前有效内容或指定版本内容。
- 隐藏编辑和修订操作，便于阅读验收。
- 支持 DOCX 导出。

验证重点：

- 与当前稿件内容一致。
- 指定版本阅读时不会误显示当前草稿内容。
- 导出文件能正常下载。

### 8.3 历史版本

路径：`/projects/:id/docs/:docType/versions`、`/projects/:id/docs/:docType/versions/:revisionId`

功能点：

- 展示 Doc 历史 Revision。
- 查看某一版本详情。
- 支持从详情进入清洁阅读。

验证重点：

- 每次提交/通过/退回产生的版本记录可追踪。
- 版本状态、操作人、时间、说明准确。

### 8.4 审稿工作台

路径：`/review`

功能点：

- 仅编辑/管理员可用，作者进入应看到无权限提示。
- 读取待审队列：`/api/review`。
- 根据 URL 参数 `docId` 优先选中指定稿件。
- 左侧展示待审稿件列表，包含项目、标题、作者、字数、提交时间。
- 右侧展示提交说明、当前稿件预览、上一轮有效内容。
- 支持填写审核反馈。
- 支持退回作者修改。
- 支持审核通过。
- 可跳转详细 Doc 审稿流。

验证重点：

- 审核通过/退回后队列刷新，已处理稿件移除。
- 退回无反馈时应阻止提交。
- 权限不足时不能执行审核接口。

## 九、管理员与项目治理

### 9.1 用户管理

路径：`/admin/users`

功能点：

- 查看用户列表。
- 新建用户。
- 查看用户详情。
- 编辑用户基础信息与角色/状态。
- 启用/禁用用户。
- 重置密码并展示临时密码。

验证重点：

- 用户状态变化后登录权限立即生效。
- 重置密码后用户需要按产品口径完成改密。
- 角色变化后导航与权限刷新。

### 9.2 注册审批

路径：`/admin/approvals`

功能点：

- 查看待审批/已处理注册申请。
- 查看申请详情。
- 审批通过。
- 拒绝申请并填写原因。

验证重点：

- 通过后用户变为 active。
- 拒绝后用户变为 rejected，并可在账号状态页看到原因。
- 审批操作产生通知/待办状态更新。

### 9.3 绑定关系

路径：`/admin/bindings`

功能点：

- 查看编辑和作者绑定关系。
- 新增绑定关系。
- 解除绑定关系。

验证重点：

- SI 适配作者、项目负责人选择应受绑定关系影响。
- 解除绑定后不应影响已存在项目的历史归属展示。

### 9.4 系统参数

路径：`/admin/params`

功能点：

- 管理 SI 主类型：新增、编辑、启用/停用。
- 管理阶段计划默认项。
- 阶段默认项用于新项目创建时初始化计划。

验证重点：

- 启用主类型应出现在 SI 新建/筛选中。
- 停用主类型不应作为新建默认可选项，但历史 SI 仍能显示原类型。
- 阶段默认项更新后，新转项目应采用新默认计划。

### 9.5 审计日志

路径：`/admin/audit`

功能点：

- 查看关键业务操作日志。
- 支持条件筛选。
- 查看审计详情。

验证重点：

- 登录、审批、用户管理、SI 转项目、Doc 流转、项目状态变更等关键操作应入审计。
- 筛选项和详情展示一致。

### 9.6 项目治理

路径：`/governance/projects`、`/governance/projects/:id`

功能点：

- 管理员查看全量项目治理列表。
- 支持搜索、阶段、生命周期、编辑、作者、逾期筛选。
- 查看治理详情。
- 调整项目编辑/作者。
- 查看或调整阶段计划。
- 管理项目生命周期：完成、归档、取消、恢复等。
- 下载或导出项目终稿。

验证重点：

- 治理列表与我的项目使用同一展示口径，但数据范围更大。
- 生命周期变化后项目详情只读状态准确。
- 分配编辑/作者后相关人员的项目列表、待办、权限同步变化。

## 十二、推荐前端验证主流程

### 12.1 账号与权限流程

1. 作者注册新账号。
2. 管理员在注册审批中通过。
3. 作者登录，确认导航只有作者入口。
4. 管理员禁用该账号。
5. 作者再次登录，应被阻止或看到禁用状态。
6. 管理员恢复/重置密码后，验证登录与改密口径。

### 12.2 SI 到项目流程

1. 编辑新建 SI 草稿。
2. 编辑保存并预发给作者。
3. 作者在“我的 SI”查看预发内容。
4. 编辑在预发记录中转项目。
5. 作者看到 converted SI 并可进入项目。
6. 编辑在项目列表看到新项目。
7. 校验通知、待办、审计是否生成。

### 12.3 Doc 阶段推进流程

1. 作者进入项目梗概 Doc，编辑并保存草稿。
2. 作者提交审核。
3. 编辑在审稿工作台看到待审稿件。
4. 编辑退回，作者在待办中看到退回待改。
5. 作者修改后重新提交。
6. 编辑审核通过，项目推进到下一阶段。
7. 重复验证细纲、正文、质检阶段。

### 12.4 正文与质检流程

1. 细纲通过后进入正文阶段。
2. 新增多个章节。
3. 调整章节顺序。
4. 删除未通过章节。
5. 作者逐章提交，编辑逐章审核通过。
6. 全部章节通过后，编辑解锁质检。
7. 进入质检 Doc，提交并审核通过。
8. 编辑标记项目完成。
9. 导出项目 DOCX。

### 12.5 管理治理流程

1. 管理员查看项目治理列表。
2. 按阶段、生命周期、编辑、作者、逾期筛选。
3. 进入治理详情。
4. 调整负责人或阶段计划。
5. 执行归档/取消/恢复等生命周期操作。
6. 回到编辑/作者账号验证项目可见性与只读状态。

## 十四、当前已知待闭环项

以下项目在页面验证时建议标记为“已识别缺口/待产品确认”，不要误判为随机回归：

| 缺口 | 当前表现 | 建议处理 |
| --- | --- | --- |
| SI 归档/删除 | `/si` 列表有按钮但全部禁用 | 明确是否需要上线；需要则接入接口、权限、二次确认、审计与列表刷新 |
| SI 版本回滚 | 版本页有回滚相关口径但前端不可用 | 若产品需要，补齐回滚确认、接口调用、冲突校验、审计 |
| 报表统计维度 | 维度选择只改变前端状态，不影响接口和图表 | 明确维度需求；接入接口参数或移除控件 |
| AI 助手 | 当前为前端演示响应 | 接真实 AI 服务前，页面文案应明确“演示/模拟”；接入后需增加权限、限流、错误处理 |
| 强制改密 | 已有账号状态口径，但需要端到端复核 | 验证重置密码后是否强制进入改密流程，必要时补服务端保护 |
| Doc 修订追踪 | 当前重点支持内容流转，结构化修订能力仍需复核 | 如需正式验收“修订接受/拒绝”，需补齐完整交互与数据模型 |
| 章节重排说明 | 章节页有禁用按钮 | 若无实际功能，建议移除；若需要，补弹窗说明或批量重排 |
| 筛选持久化 | 多数列表为本地筛选，不写入 URL | 如需验收可分享链接/刷新保留筛选，需统一 URL query 方案 |
| 分页能力 | 多数列表当前一次性加载后前端筛选 | 数据量上来后需补分页、服务端筛选与加载状态 |
